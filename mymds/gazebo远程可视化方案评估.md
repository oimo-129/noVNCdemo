# Gazebo 远程可视化方案评估

## 一、功能需求

用户通过浏览器或桌面客户端可以远程访问 Gazebo 仿真环境，无需本地安装 Gazebo，支持跨平台，交互响应延迟要在可接受范围内，最终功能模块要集成到现有Studio桌面程序中。



## 二、方案对比

### 方案 1：gzweb

![1280X1280](/home/leju/桌面/1280X1280.PNG)

**技术原理**

gzweb 是 Gazebo 官方提供的 Web 客户端，通过 WebSocket 传输 3D 场景数据（而非视频流），使用 Three.js 在浏览器中实时渲染仿真场景。

**优点评估**

- 传输的是场景数据，带宽需求低，交互延迟低
- 基于 WebSocket + Three.js，浏览器原生支持，支持多用户场景
- Gazebo 官方维护

**缺点评估**

- 渲染差异，与本体客户端使用引擎不同，机器人纹理、环境光照效果不一致，模型加载差异待排查

- 对版本依赖更严格，需特定的 Gazebo、Node.js 版本，部署及升级繁琐

- 依赖模型配置，需要同步配置 模型资产，以便于web端加载显示

  

  

### 方案 2：noVNC

![截图 2026-01-07 18-55-47](/home/leju/桌面/截图 2026-01-07 18-55-47.png)

**技术原理**：捕获 Gazebo 图形界面，使用 noVNC（VNC 的 WebSocket 代理）将画面传输到浏览器（使用视频流），用户看到的是完整的 gzclient 界面。

**优点评估**

- 无需针对 Gazebo 进行特殊开发，完整展示 gazebo 的所有功能
- 保持 web 端机器人与 gazebo 本体完全一致
- VNC 协议稳定、技术成熟，可以复用于其他图形应用的远程访问需求

**缺点评估**

- 传输的是视频流，带宽需求大，有延迟，具体延迟程度待验证
- 在复杂场景会压缩视频导致画质下降，资源消耗大



## 三、综合对比

| 对比维度 | gzweb | noVNC | 推荐 |
|---------|-------|-------|:-----|
| **性能** | 优秀 | 一般 | ✅ gzweb |
| **功能完整性** | 有限 | 完整 | ✅ noVNC |
| **用户体验** | 流畅、响应快 | 有延迟、画质受压缩影响 | ✅ gzweb |
| **服务器成本** | 低（渲染在客户端） | 高（服务器端渲染） | ✅ gzweb |
| **通用性** | 仅限 Gazebo | 可用于任何图形程序 | ✅ noVNC |

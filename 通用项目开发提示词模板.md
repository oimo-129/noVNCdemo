# 通用项目开发提示词模板

## 使用说明
将 `【变量】` 部分替换成你的具体项目需求，即可生成定制化的开发指令。

---

## 提示词模板

### 【角色定位】
你是一位资深的全栈工程师，精通 `【技术栈1】`、`【技术栈2】`、`【技术栈3】`，擅长使用 `【主要编程语言】` 解决复杂的工程问题。

### 【受众定义】
我是 `【技能水平】`（如：前端初学者/后端开发者/全栈新手），接触过 `【已知技术】`，但都不够深入，需要详细的指导。

### 【核心任务】
实现一款 `【项目描述】`。

**技术方案：**
- 前端：`【前端技术栈】`
- 后端：`【后端技术栈】`
- 核心技术：`【关键技术点】`
- 性能目标：`【性能指标】`

### 【思维链要求】
在给出实现方案前，请先一步步分析：

1. **识别项目的核心技术难点**
   - 列出 3-5 个关键挑战

2. **列出至少 3 种可行方案**
   - 对比优缺点
   - 说明为什么选择当前方案

3. **说明整体架构和数据流**
   - 描述系统各组件的交互关系
   - 绘制数据流向图（文字描述）

4. **解释技术选型理由**
   - 为什么选择这些技术组合
   - 每个技术的优势是什么

5. **列出可能遇到的坑和解决方案**
   - 常见问题预判
   - 最佳实践建议

### 【参考示例】（可选）
```
示例 1 - [功能模块名称]：
[示例代码或配置]

示例 2 - [另一个功能模块]：
[示例代码或配置]
```

### 【约束条件】
1. **执行方案前，先给我一个 Markdown 格式的任务规划清单**
   - 包含阶段划分
   - 具体任务项
   - 估计时间（可选）

2. **代码规范**
   - 所有代码必须有详细的中文注释
   - 每个函数、关键逻辑都要注释
   - 遵循 `【编码规范】`（如：PEP8/Airbnb/Google Style）

3. **解释关键技术选型的原因**
   - 技术 A vs 技术 B 的对比
   - 为什么在当前场景下更合适

4. **指出常见的坑和最佳实践**
   - 问题 1 → 解决方案
   - 问题 2 → 优化方案
   - 最佳实践清单

5. **提供完整的项目文件结构**
   - 目录树形式
   - 文件作用说明

6. **分步骤给出完整代码**
   - 环境准备
   - 核心功能开发
   - 集成测试
   - 部署上线

- `【项目描述】` → 可以多人协作的在线代码编辑器
- `【前端技术栈】` → React + Monaco Editor
- `【后端技术栈】` → Node.js + Express + Socket.io
